üîÑ Request Lifecycle

1

Request Ingestion

Client sends a POST request to /api/chat/message with the problem statement. The server creates or retrieves a ConversationState object to track the session.

2

Feature Planning Phase

The FeaturePlannerAgent analyzes requirements and proposes features. Results are streamed via SSE with type features_proposed. User can refine or approve.

3

Architecture Design

Upon approval, ArchitectAgent designs the system architecture including tech stack, folder structure, and creates a 5-batch generation plan. Emits architecture_designed event.

4

Batch Code Generation

The BatchCodeGeneratorAgent processes each batch sequentially. Each batch generates multiple files in a single LLM call, with previous batches as minimal context. Files stream as file_generated events.

5

Validation & Delivery

The BatchIntegrationValidatorAgent checks imports, types, and file coherence. Final results are packaged and can be downloaded as ZIP or executed via /api/execute.

Quick Start

Get up and running in under 5 minutes

1

Set Up Environment

Clone the repository and configure your API key

# Clone and setup
git clone <repository-url>
cd ai-coder

# Set your Gemini API key
export GEMINI_API_KEY='your-api-key-here'
2

Start the Backend

Install dependencies and run the server

cd backend
pip install -r requirements.txt
python main_enhanced.py

# Server starts at http://localhost:8000
# API docs at http://localhost:8000/docs
3

Start the Frontend (Optional)

Launch the interactive chat interface

cd frontend
npm install
npm start

# UI available at http://localhost:3000
4

Generate Your First App

Describe what you want to build in natural language

curl -X POST http://localhost:8000/api/chat/message \
  -H "Content-Type: application/json" \
  -d '{"message": "Build a todo app with user authentication"}'

API Reference

Complete API documentation for integrating with your applications

POST /api/chat/message Main chat endpoint - streaming SSE response

Send a natural language description and receive a streaming response with generated files. This endpoint handles the complete conversation lifecycle including feature planning, approval, and code generation.

Request Body
{
  "message": "string",           // Required: Description or response
  "conversation_id": "string"  // Optional: Continue existing conversation
}
SSE Event Types
// Session started
{ "type": "started", "conversation_id": "uuid" }

// Feature planning complete - awaiting user approval
{ "type": "features_proposed", "data": {
    "features": [{ "title": "...", "description": "..." }],
    "awaiting_confirmation": true
}}

// Architecture designed
{ "type": "architecture_designed", "data": {
    "project_type": "fullstack",
    "tech_stack": { "frontend": "React", "backend": "FastAPI" },
    "batch_count": 5
}}

// Individual file generated
{ "type": "file_generated", "data": {
    "filepath": "src/App.tsx",
    "content": "...",
    "language": "typescript"
}}

// Generation complete
{ "type": "code_generated", "data": {
    "total_files": 35,
    "batches": 5
}}

// Stream end
{ "type": "message_end" }
